# Cursor Working Rules

## 1. 작업 원칙
- 최소 수정 우선 (기존 구조 유지)
- 새 기능보다 기존 문제 해결 우선
- 추정 금지, 반드시 코드/로그 기반으로 판단
- 불필요한 리팩토링 금지
- 다른 파일 영향 최소화

## 2. 코드 수정 규칙
- 수정 파일을 먼저 명시
- Before / After 형태로 변경점 제시
- 기존 로직 깨지지 않게 유지
- 타입 에러 금지 (`npx tsc --noEmit` 필수 통과)

## 3. 디버깅 규칙
- 로그는 필요한 것만 추가 (최대 3~5개)
- 원인 파악 후 디버그 로그 제거
- "가능성"이 아니라 "확정 원인"만 보고

## 4. 출력 형식 (항상 동일)
1. 원인
2. 수정 파일
3. Before / After 코드
4. 재검증 결과
5. `npx tsc --noEmit` 결과

## 5. 금지 사항
- 장황한 설명
- 과거 대화 반복
- 불필요한 코드 생성
- 여러 해결안 제시 (하나로 확정)
- "추정", "아마", "가능성" 표현

## 6. UX 원칙
- 현장 사용성 최우선
- 클릭/입력 최소화
- 로그인/인증 단순화
- 사용자 혼란 금지

## 7. 우선순위
1. 사용자 요구
2. 버그 해결
3. 단순성
4. 확장성
