# Project State

## 1. 인증 / 세션
- 세션은 하나만 사용: `autoflow_user_v1`
- 로그인: 이름 입력만
- 로그아웃: 모든 세션 삭제 후 `/login` 이동
- front-ops는 별도 로그인 없음 (공통 세션 사용)

## 2. 기본 흐름
- 로그인 → `/chat`
- 미로그인 → `/login`
- front-ops 접근 시 로그인 없으면 리다이렉트

## 3. 사용자 구조
- 사용자 식별: name 기반
- DB user_id는 선택적 (UUID 있을 때만 사용)
- actor_name = 실제 작업자 이름

## 4. front-ops 정책
- 카드 생성 기준: 최근 48시간 모바일 메시지
- showDone=false 기본
- 작업자는 공통 로그인 이름 사용
- 별도 입장 화면 없음

## 5. 금지 정책
- "u-front" 같은 가짜 user_id 금지
- UUID 컬럼에 문자열 삽입 금지
- 세션 2개 이상 사용 금지

## 6. 현재 완료 상태
- 로그인 구조 통합 완료
- front-ops 정상 동작
- 모바일 흐름 정상
- 로그아웃 문제 해결

## 7. 다음 작업
